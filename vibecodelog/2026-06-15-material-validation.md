# 2026-06-15 素材校验与错误提示日志

## 本次做了什么

- 在 dashboard 渲染入口新增素材合法性检查。
- 新增 `DashboardWindow::validateConfig()`，在启动渲染前统一校验配置。
- 支持检查：
  - 背景、音乐、歌词、输出路径是否为空
  - 素材文件是否存在
  - 素材文件是否可读
  - 素材文件是否为空
  - 背景、音乐、歌词扩展名是否属于支持格式
  - 歌词是否疑似缺少 LRC 时间轴标签
  - 输出路径是否是 `.mp4`
  - 输出目录是否存在
  - 输出文件是否需要覆盖
- 输出路径未带 `.mp4` 时会自动补全。
- 输出目录不存在时可由用户确认后自动创建。
- 已更新 `README.md` 的渲染前检查说明。

## 用户体验改进

- 以前素材错误通常会在渲染线程中失败。
- 现在会在点击渲染后立即弹出明确提示，减少等待和无效渲染。
- 对可能不是标准 LRC 的歌词文件给出继续/取消选择，而不是直接中断。

## 验证结果

- 执行 `cmake --build build -j` 编译成功。
- VS Code 问题面板未发现错误。
